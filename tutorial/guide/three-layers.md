# 三层编码

invnz 的核心思想是 **三层编码**。从我们熟悉的拼音出发，经过两层抽象，最终抵达形似汉字的音韵字。

```
          抽象层次递增 →
 拼音 (pinyin)  →  音韵码 (invenma)  →  音韵字 (invnz)
  ────────      ─────────────       ──────────
  人类读写       程序处理             视觉表达
```

## 第一层：标准拼音（Pinyin）

我们熟悉的汉语拼音，如 `zhōng`、`tiān`、`guāng`。这是人类最自然的读写形式。

**但在程序眼中**，拼音有几个"不完美"的地方：

- **长度不固定**：`a` 只有 1 个字母，`zhuang` 有 6 个
- **韵母结构不显式**：`ian` 是一个整体，但内部包含介音 `i` 和韵尾 `an`
- **零声母不统一**：`a`、`ya`、`wa` 的声母在哪里？

## 第二层：音韵码（Invenma）

**音韵码** 将每个音节拆解为声母 + 韵母音素，用 `_` 分隔。每个音素都是最小单位，不可再分。

### 编码规则

```text
zhōng → zh + ong        → zh_ong
tiān  → t + ie + an     → tie_an
guāng → g + u + ang     → gu_ang
yǒng  → iong            → iong          (零声母，整体)
```

音韵码的每个部分对应一个**音素**（phoneme）：
- **声母段**：`zh`、`t`、`g` 等（与拼音一致）
- **韵母段**：分解为多个音素，如 `i-e-an`、`u-ang` 等

> 韵母的分解规则由 `vowel2phonemesMap` 定义，详见[音韵码详解](/guide/invenma)。

### 为什么需要音韵码？

音韵码让音节的结构变得**显式且可编程**：

```
zhuang（6 个字母，意义模糊）
  ↓
zh_u_ang（3 个音素，结构清晰）
```

有了音韵码，程序可以：
- 精确知道声母和韵母的分界
- 独立处理每个音素
- 为每个音素映射对应的字形

## 第三层：音韵字（Invnz）

**音韵字** 是 invnz 的终极形态——把音韵码中的每个音素，映射到一个汉字部首或组件，然后用 **汉字结构指示符（IDC）** 将它们组合起来。

结果是一个**看起来像汉字、实际上表达拼音**的字符串。

### 从音素到字形

每个音素对应一个"字根"：

| 音素 | 字根 | 说明 |
|------|------|------|
| `zh` | `⿱止` | 止，上加结构符 |
| `ong` | `工` | 工字 |
| `t` | `⿰七` | 七在左 |
| `ie` | `也` | 也字 |
| `an` | `⿵𰃦` | 冂变体，包在上 |
| `g` | `⿰𠂎` | 𠂎在左 |
| `u` | `⿱五` | 五在上 |
| `ang` | `⿹勹` | 勹包在右上 |

### 组合规则

1. **韵部内**：韵尾（en/an/eng/ang）包住前一个音素字根
2. **声母包裹**：声母的 IDC 包裹整个韵部
3. **零声母**：特定条目加 `⿰丨` 前缀

### 组合示例

```
zh + ong
  ↓        lookup(zh) = ⿱止, lookup(ong) = 工
  ↓        声母包裹韵部
⿱止工
```

```
t + ie + an
  ↓        lookup(t) = ⿰七, lookup(ie) = 也, lookup(an) = ⿵𰃦
  ↓        韵尾𰃦包也 → ⿵𰃦也
  ↓        声母⿰七包⿵𰃦也 → ⿰七⿵𰃦也
⿰七⿵𰃦也
```

## 三层对照表

| 拼音 | 音韵码 | 音韵字 | 结构解析 |
|------|--------|--------|----------|
| a | a | ⿰丨八 | 零声母 + 八 |
| ba | b_a | ⿱不八 | 不包八 |
| ban | b_an | ⿱不𰃦 | 不包𰃦 |
| bian | bie_an | ⿱不⿵𰃦也 | 韵尾𰃦包也，不包整体 |
| zhong | zh_ong | ⿱止工 | 止包工 |
| guan | gu_an | ⿰𠂎⿵𰃦五 | 𰃦包五，𠂎包整体 |
| tian | tie_an | ⿰七⿵𰃦也 | 𰃦包也，七包整体 |
| yong | iong | 用 | 整体象形 |
| wo | uo | ⿰丨火 | 零声母 + 火 |

## 设计哲学

三层编码的设计遵循一个核心原则：**每层解决一个特定的问题**。

- **拼音** 解决"人类怎么读"
- **音韵码** 解决"程序怎么拆"
- **音韵字** 解决"结构怎么看"

每一层都是下一层的"编译目标"，层与层之间的映射是**确定的、可逆的**。

```text
pinyin  ←──→  invenma  ←──→  invenz
  确定性映射       确定性映射      双向可逆
```

这种分层设计，让 invnz 既适合程序处理（音韵码），又具有视觉美感（音韵字）。
